Technical approach
Data Normalization Strategy
Your question: "If one platform provides odds as prices (0.62) and another as percentages (62%), how would you normalize and store this data?"
01
Ingest
Collect raw values from each source as-is. No transformation yet — preserve the original for audit.
02
Normalize
Convert everything to float 0–1 at ingestion layer. 62% → 0.62, 0.62 stays as-is.
03
Store
FLOAT in Postgres + source_format column for full traceability.
04
Display
Render as % in UI. One source of truth, always consistent across all markets.
// ingestion/normalise.ts function toProbability(raw: number, format: 'decimal' | 'percentage'): number { return format === 'percentage' ? raw / 100 : raw; // always 0–1 } // Polymarket returns 0.62 → toProbability(0.62, 'decimal') → 0.62 // Kalshi returns 62 → toProbability(62, 'percentage') → 0.62 // schema type MarketSnapshot = { market_id: string; source: 'polymarket' | 'kalshi'; probability: number; // FLOAT 0–1, single truth raw_value: number; // original value for audit source_format: string; // 'decimal' | 'percentage' fetched_at: Date; };
